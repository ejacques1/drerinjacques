Happens when the same repo gets imported into Vercel more than once. They're the same site with different URLs.

Copy → paste into Claude

👉 To delete a duplicate: open it in Vercel → Settings → scroll to the very bottom → Delete Project. Safe. It never touches your code or your GitHub repo.
My site works on my computer but the login/database is broken live

Your Supabase keys live in .env.local, which is (correctly) never uploaded. So Vercel doesn't have them.

Copy → paste into Claude
Do I have Supabase set up? Check whether this project has Supabase keys in a local .env file, whether the Supabase client is installed, and whether any code references Supabase. Then check whether those same keys are set on my Vercel project. Tell me: have I set Supabase up and just never added the keys to Vercel, or have I never set it up at all? Don't change anything.
👉 If the keys are only local, add them in Vercel → your project → Settings → Environment Variables. Open .env.local, and for each line the part before the = is the Key, the part after is the Value. Check Production, Preview, and Development. Then redeploy, env vars only apply to new deployments.

My login redirects to localhost instead of my real site

Two things control this, and Claude can only fix one of them.

Copy → paste into Claude
After someone signs up or confirms their email, they get redirected to localhost instead of my live site. Fix the redirect in my code to point at my live URL, then tell me exactly what I need to change in my Supabase dashboard settings to allow it. Then commit and push so it goes live.
👉 You must also update Supabase: Authentication → URL Configuration. Set Site URL to your live URL, and add your live URL to the Redirect URLs allow-list (keep http://localhost:3000/** so local work still runs). Supabase silently ignores any redirect that isn't on that list.

🪟 Windows only

After installing, it says "claude is not recognized"

The install worked, Windows just can't find the command yet. In the same terminal, paste this and press Enter (nothing visible happens, that's normal), then type claude again:

Paste into the terminal (PowerShell)
$env:Path = "$env:USERPROFILE\.local\bin;" + $env:Path
👉 To fix it permanently so this never happens again, paste this once, then open a fresh terminal:
Paste into the terminal (one time)
[Environment]::SetEnvironmentVariable("Path", [Environment]::GetEnvironmentVariable("Path","User") + ";$env:USERPROFILE\.local\bin", "User")
The install command from the video won't work ("bash is not recognized")

That command is the Mac version. On Windows (PowerShell), use this one instead, in the same terminal:

Paste into the terminal (PowerShell)
irm https://claude.ai/install.ps1 | iex
👉 If it's blocked by an "execution policy" error, run this first, then re-run the install:
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
The Claude chat panel can't see Supabase, but the terminal says Connected
The chat panel and the terminal are separate. If the terminal's /mcp shows supabase connected but the chat can't see it, close VS Code completely and reopen it (or Ctrl+Shift+P → "Developer: Reload Window"). Then start a fresh chat and try again.
The Supabase login popup won't open, or shows "connection error" (Windows)
When you run /mcp → Authenticate, Ctrl+click the link in the terminal to open it. After you approve in the browser, if it shows a "connection error" page, that's normal, copy the full URL from the browser's address bar and paste it back at the URL > prompt in the terminal, then Enter. If it says "unrecognized client ID," press Esc and run /mcp Authenticate again, logged into the Supabase account that owns your project.
The one rule that prevents most of this: a change on your computer is not a change on your site. If you built it and can't see it live, you almost certainly haven't pushed it.
