Troubleshoot & FAQ

VS Code won't install or open (Mac)
Skip Universal and download the one that matches your Mac: Apple Silicon (M1/M2/M3) or Intel. Not sure which? Apple menu → About This Mac → look at "Chip."
I don't see the Claude sunburst / icon
Open the Extensions panel (four squares) and click Claude Code. If it shows an Enable or Enable (Workspace) button, click it, a disabled extension won't show its icon. Then the sunburst appears in the left sidebar on its own (you don't need a file open). If asked to trust the folder or reload, say yes.
It says a command is "not recognized" or "not found"
Just tell Claude: "Restart the terminal and try again." This usually happens right after Claude installs something new, and a quick restart fixes it. Claude handles it, you don't need to type anything yourself.
Claude keeps asking me to approve things
That's normal and safe. Just say Yes / Approve to everything it asks so it can keep working.
An install looks frozen or is taking forever
That's normal, especially on Mac. Let it run and don't touch it. Big installs (like Node) can take a few minutes.
A browser window opened asking me to log in
That's Vercel. Log in (or sign up, it's free) to your Vercel account, then come back to VS Code. Claude will keep going.
I got a security warning opening the app
Mac: "are you sure you want to open it?" → click Open. Windows: "Windows protected your PC" → click More info → Run anyway.
It finished but I don't see my URL
Ask Claude: "Did it deploy? Give me the live URL." It will hand you the link.
Do I have to pay for anything?
Just your Claude Code subscription. VS Code, a Vercel account (free tier), and everything Claude installs are all free.
What is Node.js, and why does Claude install it?
Node.js is the software that lets code run directly on your computer instead of only inside a web browser. The Vercel deploy tool is built on it, so it's the engine that tool needs to run. Most computers don't have it by default, so Claude installs it for you when it's missing.
What is a CLI?
CLI stands for Command Line Interface, which just means controlling a program by typing commands instead of clicking buttons. The "Vercel CLI" is the typed-command version of Vercel that Claude uses to push your app live. Same Vercel, just the version a computer can operate by command.
