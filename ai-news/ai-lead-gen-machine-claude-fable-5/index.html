Build an AI Lead-Gen Machine in a Weekend with Claude Fable 5

A contractor mails every fenceless home in a zip code a postcard showing their own yard with a brand-new fence — and a QR code to book the job. Here's the exact stack and the exact prompt.

Key Takeaway

With Claude Fable 5 — Anthropic's most powerful public model, released June 9, 2026 — you can build a complete lead-generation app in a weekend. It scans aerial imagery to find homes that need your service, renders a photorealistic "after" image of each property, and mails a personalized postcard with a booking QR code, all from a single prompt in Claude Code.

Let me show you what's now possible. Imagine a contractor who could mail every fenceless home in a zip code a postcard showing their own backyard — with a brand-new fence already on it — and a QR code to book the job. A year ago, that was a six-figure software project. This week, it's a weekend.

I saw a version of this build making the rounds on X, and it stopped my scroll. So I broke it down into the exact stack and the exact prompts you'd paste in. Here's how it works.

What's in the stack — and what does each piece do?

Six tools do all the work. In plain English:

  • GitHub — holds your code.
  • Vercel — runs the app and the weekly automation.
  • Supabase — your database (stores addresses, images, and lead status).
  • Claude API — looks at aerial photos and spots the homes that need your service.
  • An image-generation API — creates the "after" picture of their property.
  • Lob — prints and mails the postcards automatically.

You don't have to understand how each one works under the hood. Fable 5 does the wiring — you just bring the accounts.

What exactly do you paste into Claude Code?

Two prompts. Use the first if you want the fence-business example as-is. Use the second for any home-service business — just fill in the brackets. Open Claude Code, paste, and let Fable 5 build.

Prompt A — The fence-business example

Build a lead-generation web app with Next.js on Vercel, Supabase, and GitHub. I enter a zip code, and the app: (1) pulls residential addresses from that county's public parcel/assessor data, (2) fetches an aerial image for each address from county GIS orthophotos, (3) uses Claude's vision API to flag homes with a yard but no fence, (4) for each qualified home, uses an image-generation API to render that yard with a new fence, (5) generates a unique QR code per address linking to a landing page showing their rendered image and a booking button, and (6) sends a postcard with the render and QR code via the Lob API. Include an admin dashboard where I review flagged homes and approve/reject before mailing. Store everything in Supabase with a status pipeline (found → qualified → rendered → approved → mailed).

Prompt B — For ANY home-service business

I run a [TYPE OF HOME SERVICE BUSINESS — e.g., landscaping, roofing, pool installation, exterior lighting]. Build me a lead-generation web app (Next.js on Vercel, Supabase, GitHub) where I enter a zip code and the app: (1) pulls residential addresses from public parcel data, (2) fetches an aerial image of each property, (3) uses AI vision to find homes that show [THE VISUAL SIGN THEY NEED MY SERVICE — e.g., a patchy lawn, aging roof, dark unlit yard], (4) renders an "after" image of that property with [MY SERVICE RESULT — e.g., fresh landscaping, new roof, night lighting], (5) creates a unique QR code per home linking to a page with their before/after images and a booking button, and (6) mails a postcard via the Lob API. Include an approval dashboard so I review leads before anything is mailed.

⚠️

One important caveat: Start with free county GIS imagery, not Google Maps. Google's terms don't allow reprinting their map images on marketing mailers — county orthophotos are public and built for this.

Dr. Erin's Analysis

The hard part of this was never the code — it's the orchestration: pulling parcel data, scoring images with vision, generating a render, and triggering a print job, all in sequence with a human approval step in the middle. That glue used to require a developer. Fable 5 writes it. Don't try to boil the ocean: start with one zip code and one service, watch the approval dashboard fill up, and only scale once the postcards are actually booking jobs.
