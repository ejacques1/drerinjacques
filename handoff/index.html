=== KEY PROMPTS ===
GitHub auth: "Set up GitHub authentication so you can save and push my work for me. Install whatever is needed, walk me through logging in, make all reasonable choices yourself, and don't stop to ask me questions. Tell me when I'm authenticated."
Build landing page: "Build me a simple landing page as an index.html web app for a [your expertise]. Fill everything in based on what you know about me, and don't ask me any questions, just build it."
Deploy: "Deploy this to Vercel. If I don't have Node or the Vercel CLI installed, install whatever is needed first. Make all reasonable choices yourself, don't stop to ask me what to do, and when you're finished, give me the live URL."
Create .mcp.json: "Create a file named .mcp.json in this project that connects to the Supabase MCP server at https://mcp.supabase.com/mcp, scoped to my project reference YOUR_REFERENCE_ID, with read/write access."
Install claude CLI (Mac): curl -fsSL https://claude.ai/install.sh | bash
Install claude CLI (Windows PowerShell): irm https://claude.ai/install.ps1 | iex
Build login: "Add basic authentication to my learning platform using Supabase Auth. I want a simple sign-up and login form with email and password, plus a logout button. Use the Supabase connection we already set up. When someone is logged out, show the login/sign-up form. When they're logged in, hide it and show a logout button with a welcome message."

=== TWO SEPARATE SUPABASE CONNECTIONS (a common confusion) ===
1. MCP (Claude <-> Supabase): lets CLAUDE build tables / run SQL. Set up via the terminal (/mcp Authenticate).
2. Keys in the page/Vercel (site <-> Supabase): SUPABASE_URL + anon key in index.html, plus keys in Vercel. This is what makes the LOGIN work in the browser.
Both are needed. One being done does not mean the other is.

=== WINDOWS GOTCHAS (this is where most calls go sideways) ===
- Install command is different: Windows uses  irm https://claude.ai/install.ps1 | iex  (not the Mac curl | bash).
- "claude is not recognized" after installing = a PATH problem, NOT a failed install. Fix in the terminal:  $env:Path = "$env:USERPROFILE\.local\bin;" + $env:Path   then type claude. Permanent fix and more are on drerinjacques.com/stuck (Windows section).
- Slash commands like /mcp only work INSIDE claude (after it launches), not at the PS C:\> prompt. Make sure they're in the Claude terminal tab, not a plain PowerShell tab.
- During /mcp Authenticate: Ctrl+click the browser link; if the redirect shows a connection error, paste the browser's address-bar URL back at the "URL >" prompt.
- If the Claude CHAT panel can't see Supabase but the terminal says Connected: fully close VS Code and reopen it.
- Quit VS Code on Windows with File > Exit (no "Code" menu).

=== OTHER KNOWN MESSES ===
- Duplicate Vercel projects: re-importing the same repo makes a new project each time. Tell: same repo, same commit, random suffixes (-b65p). Keep the one Vercel detected as the right framework; delete the rest (Project > Settings > bottom > Delete Project — safe, doesn't touch code).
- Vercel Supabase Marketplace: choose "Link existing account / Connect Account," NEVER "Create New Project" (that makes an empty database).
- The rule behind most "it's broken" reports: a change on your computer is not a change on your site. If it works locally but not live, it wasn't pushed.
