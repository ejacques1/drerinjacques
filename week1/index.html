1 Accounts + Repo

  • Create your GitHub account
  • Create the repository
  • Add the README
Done by hand on github.com, no prompts.
✓ Done when: the repo exists on GitHub with a README.

2 Set Up Your Workspace

  • Install VS Code
  • Install GitHub Desktop, sign into GitHub
  • Clone the project to your computer
  • Open the project in VS Code
  • Install the Claude Code extension and sign in
Done by hand, no prompts.
👉 Windows: quit VS Code with File → Exit (there's no "Code" menu).
✓ Done when: the Claude sunburst opens in VS Code and responds.

3 Build + Deploy

  • GitHub authentication (the gh login)
  • Build your landing page
  • Set Claude to commit directly to main
  • Commit + push
  • Deploy to Vercel
GitHub authentication
Set up GitHub authentication so you can save and push my work for me. Install whatever is needed, walk me through logging in, make all reasonable choices yourself, and don't stop to ask me questions. Tell me when I'm authenticated.
Build the landing page
Build me a simple landing page as an index.html web app for a [your expertise]. Fill everything in based on what you know about me, and don't ask me any questions, just build it.
Deploy to Vercel
Deploy this to Vercel. If I don't have Node or the Vercel CLI installed, install whatever is needed first. Make all reasonable choices yourself, don't stop to ask me what to do, and when you're finished, give me the live URL.
👉 Import your repo into Vercel once. Re-importing on a failed deploy creates duplicate projects.
✓ Done when: your landing page is live on a Vercel URL.

4 Give Your App a Database (Supabase)

  • Create a Supabase project → copy your Project Reference ID
  • Have Claude create the .mcp.json (prompt below)
  • In the terminal: install the Claude CLI → run claude → sign in → trust folder
  • Run /mcp → Authenticate → Connected
  • Vercel → Integrations → Marketplace → Supabase → Install → Link existing account → connect project
  • Vercel → Environment Variables → confirm the Supabase keys
Create the .mcp.json (swap in your Reference ID)
Create a file named .mcp.json in this project that connects to the Supabase MCP server at https://mcp.supabase.com/mcp, scoped to my project reference YOUR_REFERENCE_ID, with read/write access.
Install the Claude CLI — Mac (terminal)
curl -fsSL https://claude.ai/install.sh | bash
Install the Claude CLI — Windows PowerShell (terminal)
irm https://claude.ai/install.ps1 | iex
👉 Two separate connections here: the MCP (Claude ↔ Supabase, for building tables) and the keys in Vercel (your live site ↔ Supabase). Both need to be done.
👉 Windows: if the terminal says claude is not recognized after installing, it's a PATH issue, see drerinjacques.com/stuck (Windows section).
✓ Done when: Claude can list/create tables AND the Supabase keys show in Vercel.

5 Build Your Login

  • One prompt builds sign up / log in / log out
  • Fill in SUPABASE_URL + the anon/public key in index.html (never the service_role key)
  • Sign up, log in, and log out yourself
Build the login
Add basic authentication to my learning platform using Supabase Auth. I want a simple sign-up and login form with email and password, plus a logout button. Use the Supabase connection we already set up. When someone is logged out, show the login/sign-up form. When they're logged in, hide it and show a logout button with a welcome message.
👉 The login won't work until SUPABASE_URL + the anon key are in the page. Get them from Supabase → Settings → API. The anon key is public, so it's safe in the page.
👉 Test it locally first, then commit + push to make it live.
✓ Done when: your test account appears in Supabase → Authentication → Users.
